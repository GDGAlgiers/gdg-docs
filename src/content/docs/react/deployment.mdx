# Introduction

Deploying a React application requires choosing the right hosting platform, managing environment variables, and optimizing builds for production. This guide explores best practices for deploying React apps to **Vercel, Netlify, and Azure**, along with techniques for optimizing performance.

## Hosting Options: Vercel, Netlify, and Azure

### **Deploying to Vercel**
Vercel is a popular hosting platform for React apps, especially Next.js projects.

#### **Steps:**
1. Install the Vercel CLI:
   ```sh
   npm install -g vercel
   ```
2. Link your project:
   ```sh
   vercel login
   vercel
   ```
3. Follow the CLI prompts to deploy your app.
4. For continuous deployment, connect your GitHub repository to Vercel and push updates to trigger automatic deployments.

### **Deploying to Netlify**
Netlify provides an easy-to-use interface and supports automatic deployments from GitHub.

#### **Steps:**
1. Install the Netlify CLI:
   ```sh
   npm install -g netlify-cli
   ```
2. Link your project:
   ```sh
   netlify login
   netlify init
   ```
3. Deploy your app:
   ```sh
   netlify deploy --prod
   ```
4. Alternatively, connect your GitHub repository in the Netlify dashboard for automatic deployments.

âœ… **Best Practice:** Use Netlify Functions for serverless API endpoints.

### **Deploying to Azure**
Azure offers various deployment options, including **Azure App Service** and **Azure Static Web Apps**.

#### **Deploying with Azure Static Web Apps:**
1. Install the Azure CLI:
   ```sh
   npm install -g azure-functions-core-tools
   ```
2. Create an Azure Static Web App from the Azure portal.
3. Link your GitHub repository for automatic deployment.

## Build Optimization for Production

Optimizing the React build improves performance and load time.

### **Optimizations to Apply**
- **Minify JavaScript & CSS:** Webpack automatically does this in production builds.
- **Code Splitting:** Load only the necessary code per page using `React.lazy`.
- **Gzip Compression:** Enable Gzip or Brotli compression on the hosting platform.
- **Lazy Loading:** Load images and components only when needed.

### **Generating an Optimized Build**
```sh
npm run build
```
This creates a `build/` folder with optimized production-ready files.

âœ… **Best Practice:** Use Lighthouse or PageSpeed Insights to test performance after deployment.
